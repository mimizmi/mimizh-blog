## 什么是所有权

Rust 的所有权系统是其最独特的语言特性。三条核心规则：

1. Rust 中的每一个值都有一个被称为其所有者（owner）的变量
2. 值在任一时刻有且只有一个所有者
3. 当所有者离开作用域，这个值将被丢弃

这三条规则看似简单，但它们的推论构成了 Rust 内存安全的基础。

## 借用与引用

如果你不想转移所有权，可以使用引用。Rust 的引用有两种：

- 不可变引用 `&T` — 可以有多个
- 可变引用 `&mut T` — 同一时刻只能有一个

这个限制是 Rust 在编译期防止数据竞争的关键机制。在 C++ 中你需要依赖 sanitizer 和 code review 来保证同样的安全性；Rust 直接在类型系统层面解决了这个问题。

## 生命周期

生命周期标注是 Rust 所有权系统中最让人困惑的部分。但核心思想其实很直观：

> 引用的生命周期不能超过被引用值的生命周期。

当你写下 `fn foo<'a>(x: &'a str) -> &'a str`，你只是在告诉编译器：返回的引用与输入的引用活得一样长。

大多数情况下 Rust 可以自动推断生命周期。只有在跨函数边界且存在多个引用参数时，才需要手动标注。
